Finding Duplicate Data Survey

Example
You have a table of friends' names, ages, birthdays, and phone numbers. You know none of your friends share a phone number, so if you see a second row with a particular phone number, you know it refers to the same friend as the first row with that phone number. So you select the checkbox for phone number.

Alternatively, you know two of your friends have the same name, but not the same birthday, so you select the checkboxes for name and birthday. (Meaning if you see a second row that includes the same name AND the same birthday as an earlier row, you know it refers to the same person.)

Note: It's important to pick birthday, not age. Ex: You have two friends named Alice, one 37 years old, one 38. If you see [Alice, 37] and [Alice, 38], these may be separate rows for the two Alices (ex: [Alice, 37, 1-1-1980], [Alice, 38, 2-2-1979]), or they may be two rows for the older Alice that were retrieved at different times (ex: [Alice, 37, 2-2-1979], [Alice, 38, 2-2-1979]). If you select birthday, not age, you will be able to collapse these two rows when they refer to the same Alice, because they will include the same birthday. The general rule is we shouldn't include columns if the person's value for the column may change frequently -- age, current favorite book, most recent meal, so on.
